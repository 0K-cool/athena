# PTES Methodology Coverage — Should Be Dynamic Per Engagement

**Created:** 2026-03-22
**Status:** Feature request
**Priority:** MEDIUM — operator needs to see what ACTUALLY happened

## Problem

The PTES matrix is static HTML — shows the same tools/descriptions for every engagement regardless of what agents actually did. It should be dynamic, showing real tools used and techniques applied.

## Expected Behavior

The matrix cells should update in real-time based on actual agent activity:

| Phase | Static (Current) | Dynamic (Expected) |
|-------|-----------------|-------------------|
| Intel Gathering | "Naabu, Httpx, Katana" | "naabu (31 ports), nmap -sV (service detection)" — what AR actually ran |
| Threat Modeling | "CVE Research, 7+ databases" | "searchsploit (5 CVEs found), NVD lookup" — what DA actually queried |
| Exploitation | "SQLMap (read-only)" | "Metasploit (Samba CVE-2007-2447), nc (port 1524 bindshell)" — what EX used |
| Reporting | "Executive Summary, Technical Report" | "3 reports: Technical + Executive + Remediation Roadmap" — what RP generated |

## Implementation

1. Track which tools each agent calls (already tracked in `_tool_count` and events)
2. Map tool names to PTES phases
3. Update matrix cells via WebSocket when agents complete tool calls
4. Show actual tool names + hit counts instead of static descriptions

## Data Source

The `scan` records and `tool_start`/`tool_complete` events already have the tool name and agent. Aggregate by phase:
- PTES-2 (Intel Gathering): tools called by AR agent
- PTES-3 (Threat Modeling): tools called by DA agent
- PTES-4 (Vuln Analysis): tools called by WV agent
- PTES-5 (Exploitation): tools called by EX agent
- PTES-6 (Post-Exploitation): tools called by PE agent
- PTES-7 (Reporting): reports generated by RP agent
